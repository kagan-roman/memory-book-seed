# Правила стиля и вёрстки книги памяти

Обязательный системный контекст для агентов (`chronicler`, `reviewer`,
`cartographer`, `typesetter`, `book-director`, `wiki-curator`). Читать перед
работой над любым черновиком, ревью, картой, вёрсткой. Это шаблон —
`book-director` адаптирует «о ком книга» под конкретный род из
`book.config.yml`; остальное универсально и не размывается. Поверх — 17
железных правил `CLAUDE.md §2`.

## 1. Охват книги

- Книга — про фамилии из `book.config.yml → scope.surnames` (по умолчанию —
  `subject.surname`). Если материнская линия не включена
  (`scope.include_maternal:false`) — её в этой книге нет.
- Другие линии — возможная отдельная книга в будущем. Не размывать фокус.

## 2. Запрещённые слова и конструкции

### 2.1. Англицизмы, латиница, технические термины — вон из текста

`OCR`, `URL`, `http(s)://`, любые доменные имена; «фетч/фетчинг/фетч
DD.MM.YYYY», `request`, `dataset`, «база данных»; «20 тысяч футов» (мерять в
метрах); `дислокация`→расположение, `коммуникации`→связь, `эпизод`→случай,
`локация`→место, `активность`→действия, `верификация`→проверка; «**клан**»
(только семья/ветка/фамилия); «**узел**» о семьях.

### 2.2. Сокращения — расшифровывать при первом упоминании

`БВ`→«пропал без вести»; `СД`→«стрелковая дивизия» (далее допустимо
«272-я Свирская», «1061-й полк»); `СП`→«стрелковый полк»; `ВПП`→«военно-
пересыльный пункт»; `ЭГ/СЭГ`→«эвакогоспиталь/сортировочный эвакогоспиталь»;
`УПК`→«учётно-послужная карточка»; `РВК/ОВК`→«районный/областной военкомат»;
`ВКП(б)` — расшифровать при первом упоминании. Номера частей — цифрами
(«1061-й стрелковый полк»).

### 2.3. Архивно-методологические отступления — вон из основного текста

Списки вариантов написания фамилии — **один раз** в предисловии (раздел «Об
источниках»), в главах ни слова. Рабочие заметки исследователя («голосов не
найдено», «портал недоступен», «повторим позже», «искать через …») в книге
не место: факт не подтверждён — «обстоятельства неизвестны» одним
предложением. Процесс поиска по опечаткам — один раз в предисловии.

### 2.4. Внутренний монолог, эмоции, реконструкция (= железное правило §1)

Нельзя «он подумал/казалось/представил/мелькнула мысль»; «был испуган/
переживал/охватил страх» без прямого источника; «утром, когда туман ещё
висел…»; пафосные клише («Великая Родина», «несгибаемая воля», «святой
долг») — только прямой цитатой с атрибуцией.

## 3. Ссылки и сноски

В тексте глав — никаких ссылок/URL/доменов. Все источники — нумерованными
сносками `[^N]`; раздел «## Источники» в конце **главы** на стадии черновика,
а в финальной книге `typesetter` сводит всё в **единый список в конце книги**,
по главам (железное правило §9). URL для себя исследователь хранит в
`book/chapters/<slug>/` рабочей заметке, не в печатном тексте.

## 4. Структура и повторы

- Что у фамилии несколько веток и как ветки связаны — сказано **во введении**.
  В главах не повторять; достаточно «из ветки <…>». Родство с другим
  ветераном — одной фразой со ссылкой на его главу через эпитет (§16), без
  пересказа.
- Методология (опечатки фамилии, кто из родни нашёл документ) — один раз там,
  где критично; не в каждом абзаце.
- Сильный драматический кадр — **один раз**. Один раз — сильно, два —
  навязчиво, три — фальшь (= §3, §4).
- Не ссылаться на таблицы/«под номером N» исходной родословной — самих таблиц
  в книге нет, ссылка пуста и сбивает.
- Две женщины с одинаковым именем в одном поколении — различать отчеством или
  двумя датами рождения, не оставлять обеих без различения.

## 5. Карты и иллюстрации

- **Линию фронта не рисуем** (нет надёжных данных).
- Главы без боевого пути всё равно получают **карту-якорь**: место рождения,
  место призыва, возможные направления/сектора заштрихованными зонами без
  жирных линий — географический якорь, не фейковая точность.
- **Карты чередуются с текстом, не в конце.** Одна карта-якорь — после
  зачина/пролога. 2–3 карты — каждая перед своей операцией: читатель идёт по
  карте во время чтения.

## 6. Вёрстка (для typesetter)

1. Колонтитулы — в общем стиле, **не белые** «дыры».
2. Колоночность — по теме (`build/theme.css`); единообразно для глав, введения,
   заключения, врезок.
3. Текст не выезжает за поля; переполнение — перенос на следующую страницу,
   **не** автосокращение содержания. Плотную главу — снизить плотность строк,
   не резать смысл.
4. Нумерация сквозная, включая карта-листы.

## 7. Зачин и подзаголовки внутри глав (= железное правило §17)

```
# Имя Отчество. Краткий содержательный подзаголовок
<!-- tree -->            ← инлайн мини-дерево двора (без разрыва страницы)
<лёгкая проза «точка отсчёта»>

## 1. Призыв
## 2. <содержательное имя раздела>
```

**Запрещено:** «Пролог/Эпилог» как разделы; «Глава N», «Часть I» внутри
главы; голые `## 1` без названия. Имена разделов содержательные
(«Извещение», «Что было после»), не процедурные.

## 8. Финальный голос

Книга **семейная, не торжественная**. Не подарочное издание к юбилею. Цель —
сохранить имена и судьбы для потомков, которые иначе их не узнают. Тон —
спокойный, документальный, тёплый; без пафоса, без героизации, без архивных
оговорок в тексте. Читатель-потомок должен закрыть книгу с ощущением, что
**узнал предка**, а не «прочитал работу архивиста».
