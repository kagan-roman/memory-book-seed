# Правила дизайна карта-листа «Боевой путь»

Свод правил визуального контракта карт. Соблюдает `cartographer`; движок
`build_maps.py` их не нарушает. Данные одного листа — элемент массива
`book/chapters/<slug>/maps.json` (схема — `build/SCHEMA.md`).

---

## 1. Формат

- **A5 портрет**, ~874×1240 px (≈150 DPI). **Светлая печатная карта под
  кремовую бумагу книги** (не тёмная — старый тёмно-золотой макет заменён
  при переходе на Vivliostyle, см. §7). Фон карты — мягкий кремовый
  градиент под бумагу книги; halo подписей — цветом бумаги.
- Один человек = 1–N листов (по числу обособленных операций). Не пихать всю
  войну в один лист.
- Две разные операции/фронта на большом удалении — два листа в одном
  масштабе, **не один с инсетом**.

## 2. Что показывать на карте

### Точки (waypoints)

- Только в радиусе боевого участка. Точка в ~800 км от прочих (учебка в
  глубоком тылу, призывной пункт) — **не на карту**, в био-блок панели.
- Две точки в одном городе/секторе — слить (двойной лейбл).
- Нумеровать по порядку движения в пределах листа (1→N), не сквозной.

### Стрелки (военная картная нотация) — поле `arrow` сегмента

- **`combat` 🔴 красная сплошная со стрелкой** — боевое движение в составе
  части (направление наступления).
- **`redeploy` 🟥 красный пунктир с дугой** — передислокация части без боя.
  Рисуется **дугой**, не прямой через всю карту.
- **`personal` ⬜ серый пунктир без стрелки** — личное/админ. перемещение
  (призыв, госпиталь, перевод). Слабо контрастно.
- **`post` 🟢 зелёный пунктир со стрелкой** — куда дошла часть **после
  смерти** бойца. Старт — с **последней боевой позиции**, не с места
  эвакуации раненого (реальная правка: зелёная шла из госпиталя — нельзя).
  Это исторический контекст, не его маршрут.
- **`evac` серый пунктир + крестик** — медицинская эвакуация, тонкая дуга.
- **`possible` — тонкий красный пунктир, низкая насыщенность** —
  неустановленное вероятное направление (мало данных). Не выдавать за
  достоверный маршрут.

### Низкая достоверность / нет боевого пути — зоны, не жирные линии

Глава без точного пути всё равно получает **карту-якорь** (STYLE-RULES §5.2):
место рождения/призыва как контекст в панели, вероятные сектора фронта —
**штрихованными зонами** (`zones`: полигон + подпись), НЕ жирными стрелками.
Географический якорь без фейковой точности; зона в легенде — свотч `zone`.

### Линия фронта — НЕ РИСУЕМ

Надёжных датированных векторных данных линии фронта в открытых источниках нет.
«На глаз» с общих карт — неточно и путает читателя. `front_line` в конфиге не
задавать. **«Лучше нет карты, чем неправдивая карта.»**

### Города

- Крупные — из Natural Earth (`clipped_cities.geojson`), не трогать.
- Мелкие, важные для контекста, — список `extra_cities` **свой на каждый
  лист**, не общий.
- Город, упомянутый в тексте главы, **должен** быть подписан. Левых городов
  не быть. Менялось имя — «Старое (Новое)» в скобках.
- Подпись города подавлять, если в радиусе 22 px от waypoint (наезжает).

## 3. Подписи дат на стрелках

- Только на `combat` и `redeploy`. `personal`/`evac` — без подписи.
- Положение — **перпендикулярно линии стрелки**, не строго вверх (меньше
  наезда на города). Сдвиг ≥14 px от линии.
- Halo подписи перекрывает фон **полностью** (paint-order stroke), иначе
  «каша» из наложений.

## 4. Таймлайн (симметричный относительно рельсы)

```
[ТАЙТЛ таймлайна]                                              ← вверху слева
   [sub]            [sub]              [sub]                    ← sub-подписи ВЫШЕ
   призван          <город>            <операция> → †           ← главные подписи
─[спан 1]──────[спан 2]──────[спан 3]──────                     ← личные спаны, к рельсе сверху
═●═══════════════════●═══════════════════●═══════ ← РЕЛЬСА (центр, золотая)
[вся война макробоксами — к рельсе снизу, подпись внутри бокса]
  ┊       ┊       ┊       ┊                                     ← тики
 1941    1942    1943    1944    1945                           ← годы в самом низу
```

- Сверху рельсы — личные события: спаны вплотную к рельсе, точки над ними с
  лидер-линиями. Снизу — макрособытия войны: боксы вплотную к рельсе,
  подписи внутри. Годы/тики — ниже всего.
- **Точка** — короткое событие (день): точка + главная подпись + sub-подпись
  НАД главной. **Спан** — длительное (недели/месяцы): полупрозрачный бар.
- **Кластеризация:** 2+ событий в пределах 7 дней — одна метка
  («<операция> → †», 22-23.06), не подписывать каждое. Длительное — отдельный
  спан.

## 5. Правая панель (факты)

- Шапка «КОНТЕКСТ» — био 3–5 строк (откуда, когда призван, какая часть).
- Секции по фазам маршрута, цвет = цвету сегмента на карте.
- Точка панели = кружок с номером + название + дата + краткий комментарий.
- Внизу — кросс-ссылка на след. лист и источники мелким серым.

## 6. Header / Footer

- Шапка: имя крупно; eyebrow серым (полк·дивизия·армия·фронт); даты жизни и
  слайд-индикатор справа.
- Footer: **слева — `book.footer` из `book.config.yml`** (или `book.title ·
  book.subtitle`); справа — «ЛИСТ N / M · A5». Никакого хардкода рода.
- Тонкая линия-разделитель (цвет `--rule`) шапка/контент и контент/футер.

## 7. Цветовая палитра — СВЕТЛАЯ ПЕЧАТНАЯ (под кремовую книгу)

Старый тёмно-золотой макет заменён при переходе на Vivliostyle: карта
печатная, согласована с `build/theme.css` (кремовая бумага `#fffdf8`,
чернила `#1a1a1a`). Это уже зашито в `build/build_maps.py` — менять цвета
там, не «перекрашивать» отдельно.

| Назначение | Цвет |
|---|---|
| Фон карты | мягкий кремовый градиент `#fffdf8 → #fbf5ea → #f5eedd` (под бумагу книги) |
| Halo подписей (обводка) | `#fffdf8` (цвет бумаги), `paint-order:stroke`, полное перекрытие |
| Текст / приглушённый / линии-разделители | `#1a1a1a` / `#4a4a44` / `#bcae93` |
| Акцент (дом/призыв, заголовки) `TAN_HOME` | `#7a5a32` |
| Боевое `RED_COMBAT` / гибель `DEATH` | `#b5402f` / `#b23a2b` |
| Ранение/эвакуация `WOUND` | `#c07a2e` |
| После смерти `POST_GREEN` | `#5f7a45` |
| Личное/админ. `TAN_PERSONAL` | `#6b6357` |
| Возможное направление `possible` / зона `zone` | тот же `RED_COMBAT`, тонко/штрихом, низкая насыщенность |

## 8. Чего избегать

- Длинная прямая через всю карту вместо дуги для передислокации.
- Подписывать каждое плотное событие на таймлайне (кластеризовать).
- Дублировать подпись (waypoint = город → город отдельно не подписывать).
- Слово «клан». Только «семья», «ветка», «фамилия».
- Растягивать масштаб ради одной далёкой точки.
- Путать боевое движение с административным (разные стрелки, разный смысл).

## 9. Шаблонизация

Лист = словарь в `book/chapters/<slug>/maps.json` (массив на человека).
Полная схема ключей и пример — `build/SCHEMA.md`. Движок
`build/build_maps.py` переиспользуется для всех; правится только данные.

## 10. Границы карты, фейковое море, сводная карта, рассинхрон

- **`bbox` листа ⊆ обрезанных геоданных.** `clip_geo.py` режет Natural Earth
  по `book.config.yml → geo:`. Если `bbox` слайда вылезает за этот
  прямоугольник — на краях нет земли/границ → кремовый фон читается как
  «фейковое море» / «белые пятна между областями». Нужен более широкий
  обзор — расширь `geo:` и перезапусти `python3 build/clip_geo.py`, не
  тяни bbox за пределы данных. `geo:` должен покрывать **объединение всех
  bbox книги** + запас.
- **Сводная карта (заключение)** — отдельный конфиг `"kind": "overview"`
  («дороги, которые сошлись»): `origin`/`victory`/`anchor_cities`/`roads`
  (схема — `build/SCHEMA.md`). Это обзор замысла, не чей-то маршрут;
  встаёт по `<!-- map -->` в `conclusion.md`.
- **Рассинхрон карты и фактов.** Архивная перепроверка (`/research`)
  изменила даты/часть/место → **синхронно** перегенерить `maps.json` этого
  человека (`/maps --slug`) и пересобрать (`/pdf`); проверить, не задеты ли
  тем же остальные. Карта не должна противоречить главе и `facts.md`.
