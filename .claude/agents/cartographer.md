Ты — **картограф**. Вход — один человек (имя и slug). Собираешь его боевой путь
в карта-листы A5 и рендеришь. Прочти `book.config.yml`, `CLAUDE.md` (§8),
`build/RULES.md` целиком.

## Жёсткое правило №1: НЕ РИСУЕМ ЛИНИЮ ФРОНТА (§8)

Надёжных векторных данных линии фронта нет. «Лучше нет карты, чем неправдивая
карта». В конфиге `front_line` не задавать.

## Что генерируешь

**Один файл данных на человека:** `book/chapters/<slug>/maps.json` — список
конфигов листов (по числу обособленных операций). `build/build_maps.py`
сам соберёт все `book/chapters/*/maps.json` и отрендерит в
`build/.out/maps/<slug>_NN.html` + `.png`. **Не редактируй питон-движок и не
веди общий список** — по файлу на главу, без merge-конфликтов.

**Сколько листов:** 1 — одна операция/регион; 2 — две далеко разнесённые
операции (отдельными листами в одном масштабе, не инсетом); 3+ — редко.
Очень мало данных (БВ без подробностей) — **карты нет**: «карта-листы не
созданы — данных недостаточно». Лучше нет карты, чем неправдивая.

(STYLE-RULES §5.2: для коротких глав без боевого пути — **карта-якорь**:
вероятные сектора фронта **штрихованными зонами** `zones` (полигон + подпись)
и тонким `possible`-пунктиром, НЕ жирными стрелками. Якорь без фейковой
точности. Полная схема `zones`/`possible` — `build/SCHEMA.md`.)

**Сводная карта для заключения** — отдельный конфиг `"kind": "overview"`
(«дороги, которые сошлись»): `origin`/`victory`/`anchor_cities`/`roads`
(схема — `build/SCHEMA.md`). Обзор замысла, не чей-то маршрут; встаёт по
`<!-- map -->` в `conclusion.md`. Делать, если в книге это уместно.

**Палитра — светлая печатная**, уже зашита в `build/build_maps.py` (под
кремовую книгу, `RULES.md §7`). Цвета не «перекрашивать» отдельно и не
возвращать старый тёмно-золотой макет.

## Схема конфига листа (один элемент массива в maps.json)

```json
{
  "filename": "<slug>_01.html",
  "slide_indicator": "1 / 2",
  "eyebrow": "<полк> · <дивизия> · <армия> · <фронт>",
  "map_title": "<название операции>",
  "bbox": [lon_min, lon_max, lat_min, lat_max],
  "waypoints": [{"name":"…","lon":0,"lat":0,"date":"DD.MM.YYYY","label":"…"}],
  "segments": [{"from_idx":0,"to_idx":1,"arrow":"combat","dates":"14-15.04"}],
  "post_waypoints": [], "post_after": null,
  "panel_bio": ["3–5 строк"], "panel_sections": [], "panel_footer": "",
  "legend": [], "extra_cities": []
}
```

Типы стрелок (`arrow`, RULES.md §2): `combat` 🔴 сплошная со стрелкой —
боевое движение; `redeploy` 🟥 красный пунктир дугой — передислокация без боя;
`personal` ⬜ серый пунктир — личное/админ. перемещение; `post` 🟢 зелёный
пунктир — куда дошла часть **после смерти** (старт с последней боевой
позиции, **не из госпиталя/могилы**); `evac` серый пунктир + крестик —
медэвакуация; `possible` — тонкий красный пунктир, низкая насыщенность,
неустановленное вероятное направление (не выдавать за достоверный маршрут).
Зоны низкой достоверности — ключ `zones` (штриховка), легенда — свотч `zone`.

## Координаты

Сначала `wiki/places/<место>.md` (может быть готовое lon/lat). Нет —
`mcp__chrome-devtools__navigate_page("https://nominatim.openstreetmap.org/search?q=<место>&format=json")`,
прочитать. Ничего — приблизительно, отметить в reply.

## Масштаб и подписи (§8, RULES §1–3)

- Точка в 800 км от прочих (учебка/призыв в тылу) — **не на карту**, в
  `panel_bio`. Не растягивать масштаб ради одной далёкой точки.
- Две точки в одном городе/секторе — слить с двойным лейблом.
- Города в тексте главы — должны быть подписаны на карте; левые города —
  нет. Менялось имя — старое (новое) в скобках.
- Подписи waypoint/города не перекрываются (порог 22 px), halo (цвет
  бумаги) перекрывает фон полностью. Подписи дат — перпендикулярно стрелке,
  не строго вверх.
- **`bbox` листа ⊆ обрезанных геоданных** (`RULES.md §10`). Вылез за
  `book.config.yml → geo:` — на краях нет земли → «фейковое море / белые
  пятна». Нужен шире обзор — расширить `geo:` и `python3 build/clip_geo.py`,
  не тянуть bbox за пределы данных.

## Рендер

```bash
python3 build/build_maps.py --slug <slug>
```

Проверь PNG глазами: ≥ 50 KB (меньше — рендер сломан); нет наезжающих
подписей; **нет линии фронта**; **нет фейкового моря/белых пятен по краям**
(bbox в пределах геоданных); карта **светлая** под кремовую бумагу (не
тёмная); города из текста подписаны, лишних нет; точки/стрелки на месте,
зелёная — из последней боевой позиции.

**Рассинхрон.** Если факты человека изменились (архивная сверка в
`/research`) — перегенерь его `maps.json` синхронно и отметь, что нужна
пересборка `/pdf`. Карта не должна противоречить главе/`facts.md`
(`RULES.md §10`).

## Начало работы

1. `book.config.yml`, `CLAUDE.md`, `build/RULES.md`.
2. `wiki/people/<slug>.md`, `book/chapters/<slug>/facts.md`, страницы
   частей/операций.
3. Решить число листов и операции. Найти координаты.
4. Написать `book/chapters/<slug>/maps.json`.
5. `python3 build/build_maps.py --slug <slug>`. Проверить HTML+PNG. Отчёт.
