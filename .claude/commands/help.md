Покажи пользователю краткую справку (прочитав `book.config.yml`, чтобы
подсказать, на каком он шаге):

**Что это:** seed-конвейер книги памяти о предках-фронтовиках. Минимум на
входе — фамилия и село. Итог — `dist/book.pdf`.

**Команды (стадии конвейера):**
- `/start` — онбординг: вопросы → `book.config.yml` → (в авто-режиме) весь
  конвейер сам.
- `/status` — где что стоит, что делать дальше.
- `/ingest` — скан родословной → текст → список людей → черновики wiki.
- `/genealogy` — ветки, дворы, эпитеты, конфликты дерева.
- `/research [slug]` — архивы: факты + голоса сослуживцев + сканы документов.
- `/conceive` — замысел книги, введение, заключение, врезки.
- `/write [slug]` — главы (Алексиевич/Гранин, ноль вымысла).
- `/review [slug]` — по главе: факты, стиль, внутриглавные повторы, динамика.
- `/edit` — сквозная редактура всей книги (межглавные повторы, согласованность).
- `/maps [slug]` — карты боевого пути (без линии фронта).
- `/audit` — приборная панель: KPI + вердикт готовности (мягкий гейт).
- `/illustrate [propose|apply]` — документальные фото → ваш отбор → вставка.
- `/pdf` — собрать книгу в PDF (вёрстка; только по явной команде).
- `/proof` — вычитка: комментарии человека в PDF → точечные правки в главы.
- `/wiki [A|B|C]` — синхронизация знания (идёт непрерывно по ходу конвейера).

**Принципы (полностью — `CLAUDE.md §2`):** ничего не выдумываем; «семейной
памяти» нет без строки-источника; без драмы и повторов; карты без линии
фронта; автор-вступление по интервью; пересборка PDF — только по команде.

**Карты:** один раз `python3 build/clip_geo.py` (геоданные).

Подскажи конкретный следующий шаг исходя из состояния (как `/status`, кратко).
