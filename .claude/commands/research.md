Стадия **research** — фундамент книги. Прочитай `book.config.yml` и
`CLAUDE.md §5` (опечатки) и **`§6` целиком** (протокол сбора).

Для каждого человека (или `$ARGUMENTS`) запусти агента `researcher` →
`book/chapters/<slug>/facts.md` + `context.md` + скачанные сканы в
`assets/documents/<slug>/` + `MANIFEST.md`.

- Батчами по `run.parallel_batch`, по одному человеку на агента.
- Протокол §6.2 ОБЯЗАТЕЛЕН на каждого, не «по верхам»: все варианты фамилии →
  «ПОКАЗАТЬ ЕЩЁ» → каждый документ открыт → все его страницы пролистаны
  взад-вперёд (счётчик >2000 = чужая подшивка) → сканы скачаны и разложены.
  `run.depth` — это широта (сколько вариантов/перекрёстных порталов/заходов
  сверки), а не «открывать ли документы» (открывать всегда).
- Капча / 503 / гео-фенс — передай пользователю просьбу открыть у себя и
  вставить; браузер не закрывать посреди задачи (§6.1). Не блокируй конвейер
  по другим людям.
- **Проверь вывод каждого `researcher`** перед «готово» (§6.4): не приписана
  ли часть/награда с соседнего чужого скана; «(2)» = документы, не медали;
  номера части/армии/партбилета сверены; родство однофамильцев подтверждено
  отчеством+годом+селом; конфликты — обе версии; перепроверки append-only
  блоком `## АРХИВНАЯ СВЕРКА <дата>`.

Сводка: по кому богатый материал и сколько документов скачано, по кому
пробелы, какие варианты фамилии сработали, что осталось докопать/перепроверить.
Ничего не выдумывать; «источников не найдено» — нормальная запись.
