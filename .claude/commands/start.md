Ты — оркестратор seed-проекта «Книга памяти». Пользователь хочет создать книгу
о своих предках-фронтовиках. Сделай онбординг.

## 1. Прочитай контекст

Прочитай `book.config.yml` и `CLAUDE.md`. Если `subject.surname` уже заполнен —
спроси, продолжаем с текущим конфигом или перенастраиваем.

## 2. Задай вопросы (используй инструмент AskUserQuestion, по 1–4 за раз)

Минимально необходимое — **фамилия и село**. Спрашивай дружелюбно, объясняя
зачем. Собери:

- **Фамилия** (каноническое написание) и женская форма.
- **Село**, район/округ, область/край, страна.
- **Скан родословной?** Если у пользователя есть PDF семейной/родословной
  книги — попроси положить его в `sources/` и назвать файл; иначе стартуем от
  фамилии+села по архивам (`scope.start_from_archives_only: true`).
- **Охват:** только эта фамилия или несколько; включать ли материнскую линию.
- **Автор (для раздела «Как эта книга началась»):** имя, кем приходится,
  зачем книга (своими словами), контакт (телеграм/почта) для уточнений от
  родни, писатель ли он (если нет — будет честная оговорка «я инженер, а не
  писатель»). Личные «крючки» для вступления — **только то, что назовёт сам**;
  предупреди, что ничего не выдумается.
- **Режим:** автоматический (конвейер идёт сам, останавливается на гейтах) или
  поэтапный (вы вызываете стадии вручную).
- **Глубина:** quick / standard / exhaustive (объясни компромисс время↔полнота).

## 3. Предложи варианты написания фамилии

По правилам `CLAUDE.md §5` (Г↔Ч/Б/Л, К↔Х, О↔А, У↔О) выведи вероятные
архивные опечатки фамилии и **покажи их пользователю на подтверждение/
дополнение** перед записью в `surname_variants`.

## 4. Запиши `book.config.yml`

Аккуратно, сохранив все комментарии-подсказки файла, заполни поля ответами.
Не выдумывай ничего за пользователя — пустое оставляй пустым. Покажи итог.

## 5. Геоданные

Напомни (или предложи выполнить, спросив разрешение):
`python3 build/clip_geo.py` — один раз скачать и обрезать Natural Earth по
блоку `geo:`. Подскажи поправить `geo:` bbox, если театр военных действий
явно иной.

## 6. Дальше — по режиму

- `run.mode: auto` → запусти конвейер: `/ingest` → `/genealogy` →
  `/research` → `/conceive` → `/write` → `/review` → `/edit` → `/maps` →
  `/illustrate` (PROPOSE, затем **остановись и жди отбора автора**) → после
  отбора `/illustrate` APPLY → `/typeset` → `/wiki`. Между стадиями —
  короткий отчёт. **Гейты человека (CLAUDE.md §13):** перед APPLY
  иллюстраций и перед финальным PDF —
  показывай результат и жди подтверждения. PDF пересобирай только по команде.
- `run.mode: staged` → объясни доступные команды (`/status`, `/ingest`,
  `/research`, …) и жди указаний.

Соблюдай все 17 железных правил `CLAUDE.md §2`. Не начинай тяжёлую работу,
пока конфиг не подтверждён пользователем.
