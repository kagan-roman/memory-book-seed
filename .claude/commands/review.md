Стадия **review**. Прочитай `book.config.yml`, `CLAUDE.md` (§2),
`STYLE-RULES.md`.

Для каждой готовой главы (или `$ARGUMENTS`):

1. Агент `reviewer` → `book/chapters/<slug>/review.md` (факты, стиль,
   **повторы** — отдельным проходом, выдуманная «семейная память» — искать
   особо, §2).
2. Агент `book-director` пост-ревью замысла → секция в том же `review.md`
   (соответствие outline, дубли с врезками, эпитеты, тональный баланс,
   натянутая драма §3).
3. Проход на **динамику** (book-director): где нудные перечни номеров частей/
   комбатов — что подсократить, **не теряя атмосферу и фактуру** (§10).

Затем для глав с `verdict: revise` запусти `chronicler` на правки (точечно,
**информацию не вырезать**, только повторы — §10). Перезапиши `draft.md`.
**После делегированной правки сверь `git diff`** (контракт §3): не пропала
ли фактура (`[REGRESS]`), не появились ли новые утверждения без источника,
правились ли только указанные места — иначе верни в ревью или поправь сам.
Повтори ревью при необходимости (батчами `run.parallel_batch`).

Сводка: вердикты по главам, сколько блокеров снято, что осталось. PDF не
пересобирать (§13).
