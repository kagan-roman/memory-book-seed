Стадия **сборки книги** (вёрстка). Прочитай `book.config.yml`, `CLAUDE.md`
(§9, §13), `book/_master/STYLE-RULES.md`, `build/RULES.md`.

**Дисциплина (§13):** собирай только когда пользователь явно это просит
(«пересобери», `/pdf`). Не пересобирать самовольно после правок.

1. Убедись: `build/node_modules` есть (иначе `cd build && npm ci`); карты в
   `build/.out/maps/`, деревья в `build/.out/trees/` (нет — сначала
   `python3 build/build_trees.py`; если дерева у рода нет — это нормально,
   мини-деревья просто не вставятся, см. §ниже).
2. Запусти агента `typesetter` (верстальщик) — или напрямую
   `python3 build/build_book.py`. Порядок — из `book/_master/outline.md`.
   Карты/деревья — **внутри глав** по маркерам (не в конце), зачин-канон §17,
   колонтитулы не белые, текст не выезжает, источники единым разделом в
   конце (§9). Маркер `<!-- tree -->` без данных дерева — пропускается без
   ошибки (родословная может быть неизвестна).
3. Проверки: `pdfinfo dist/book.pdf`; `pdftotext -layout` — нет сырых `[^N]`,
   доменов, `<!--`; `pdftoppm` головы/середины/хвоста — глазами.

Покажи пользователю результат (`dist/book.pdf`, число страниц, размер) и
**жди подтверждения** перед тем, как считать книгу готовой. Отчёт о проблемах,
если есть.
