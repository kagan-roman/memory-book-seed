Стадия **maps**. Прочитай `book.config.yml`, `CLAUDE.md` (§8), `build/RULES.md`.

Проверь, что геоданные есть (`assets/geo/clipped_*.geojson`). Нет — сначала
`python3 build/clip_geo.py`. `geo:` в `book.config.yml` должен покрывать
**объединение всех bbox книги** + запас, иначе на краях карт «фейковое
море/белые пятна» (`RULES.md §10`); слайд за пределами геоданных — расширь
`geo:` и перезапусти clip_geo.

Для каждого человека (или `$ARGUMENTS`) запусти агента `cartographer`:
`book/chapters/<slug>/maps.json` → `python3 build/build_maps.py --slug <slug>`
→ `build/.out/maps/<slug>_NN.html`+`.png`. Батчами по `run.parallel_batch`.
Сводная карта заключения (`kind: overview`) — отдельным конфигом, если уместно.

Жёстко: **линию фронта не рисуем** (§8); дом/призыв/село не на боевую карту;
далёкая точка (учебка в тылу) — в био-панель, не растягивать масштаб; города
из текста — подписать, старое(новое) имя; halo (цвет бумаги) перекрывает фон
полностью; зелёная `post` — из последней боевой позиции, не из госпиталя.
Мало данных — карты нет (честно), либо карта-якорь со штрихованными зонами
`zones`/тонким `possible`, не жирными стрелками (STYLE-RULES §5.2). Палитра
**светлая печатная**, уже в движке — не возвращать тёмно-золотой макет.

Проверь PNG глазами: нет наезжающих подписей, нет линии фронта, нет фейкового
моря/белых пятен по краям, карта светлая под кремовую книгу. **Рассинхрон:**
если факты менялись в `/research` — перегенери карты этих людей синхронно и
пометь, что нужна пересборка `/pdf` (`RULES.md §10`). Сводка: у кого сколько
листов, приблизительные точки, у кого карты нет и почему.
