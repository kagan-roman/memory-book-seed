# Книга памяти — seed

**Форкни → запусти `claude` → получи книгу памяти о своих предках-фронтовиках:**
обложка, карты боевого пути, документальные иллюстрации, художественно-
документальные главы, готовый PDF.

Это пустой шаблон-конвейер на [Claude Code](https://claude.com/claude-code).
Вся работа — поиск по архивам, написание, карты, вёрстка — выполняется
субагентами по обкатанным правилам. От вас нужно минимум: **фамилия и село**.

> Референс — собранная по этому конвейеру книга «Кагановы из Смирново»:
> см. `book.config.yml → reference_book_url`.

---

## Зачем

Архивы (ЦАМО / pamyat-naroda) хранят миллионы судеб, но они рассыпаны по
наградным листам, донесениям, картотекам ранений. Собрать из крупиц связную,
честную, читаемую книгу о своём роде — недели работы. Этот конвейер делает её
за один прогон, **ничего не выдумывая**: где данных нет — так и сказано.

## Что нужно установить

- **Claude Code** (CLI).
- **Node 18+** и `npx` — для вёрстки (Vivliostyle).
- **Pandoc** — markdown → HTML.
- **Poppler** (`pdfinfo`, `pdftotext`, `pdfimages`, `pdftoppm`) — PDF/сканы.
- **Python 3.10+** с `Pillow` — карты, обрезка фото.
- **Google Chrome** (или Chromium) — рендер карт и доступ к pamyat-naroda
  через chrome-devtools MCP (использует ВАШУ сеть/VPN — важно для архивов РФ).

macOS: `brew install pandoc poppler node` · `pip install Pillow`.

## Быстрый старт (самый простой путь)

```bash
git clone <ваш-форк> моя-книга && cd моя-книга
claude
```

В Claude Code:

```
/start
```

Команда задаст вопросы (фамилия, село, район/область, есть ли скан родословной,
кто вы как автор и зачем книга, режим), запишет `book.config.yml` и — если вы
выбрали автоматический режим — прогонит весь конвейер сама, останавливаясь
только там, где нужно ваше решение (отбор фотографий, финальный просмотр).

Если есть скан родословной/семейной книги — положите PDF в `sources/` **до**
`/start` и укажите его имя в ответах. Нет — конвейер стартует от фамилии+села.

Итог — `dist/book.pdf`.

## Поэтапный режим (полный контроль)

Если хотите вести процесс руками — в `book.config.yml` поставьте
`run.mode: staged` (или ответьте так в `/start`) и вызывайте стадии по очереди:

```
/ingest      скан → постраничный текст → список людей → черновики wiki
/research    архивы по каждому: факты + голоса сослуживцев
/conceive    общий замысел книги, введение, заключение, врезки
/write       главы (тон Алексиевич/Гранин, ноль вымысла)
/review      проверка фактов и стиля + ревью замысла + проход на динамику
/maps        карты боевого пути (военная нотация, без линии фронта)
/illustrate  подбор документальных фото → ваш отбор → вставка
/typeset     сборка PDF (Vivliostyle)
/wiki        перенос фактов в базу знаний
/status      где что стоит
```

Каждую стадию можно перезапускать; всё состояние — в файлах, не в памяти чата.

## Карты — один раз скачать геоданные

Карты строятся поверх Natural Earth. В репозитории их нет (тяжёлые). После
заполнения `book.config.yml` (особенно блока `geo:`) один раз выполните:

```bash
python3 build/clip_geo.py
```

Скрипт скачает Natural Earth 1:50m и обрежет по вашему театру военных действий
из `geo:`. Дальше карты строятся без интернета.

## Как это устроено

```
book.config.yml   единственный конфиг — фамилия, село, автор, регион, режим
CLAUDE.md         правила и замысел (Claude читает автоматически)
.claude/agents/   8 субагентов  ·  .claude/commands/  слэш-команды-стадии
sources/          сырьё (скан, OCR) — не меняется
wiki/             база знаний (паттерн llm-wiki Карпаты): люди/места/части/события
book/             рукопись: _master/ (замысел) · chapters/<человек>/ · illustration/
assets/           бинарники по типам: documents/<человек>/ (сканы ЦАМО —
                  первоисточник) portraits/ photos/ cover/ fonts/ geo/
build/            движки сборки + theme.css + RULES.md  (генерат → build/.out/)
dist/book.pdf     итог
```

Подробно — в `CLAUDE.md` (в т.ч. **17 железных правил**: ноль вымысла, без
драмы, без повторов, карты без линии фронта, автор-вступление по интервью…).

## Честность и приватность

- Книга помечается как написанная с помощью ИИ, с контактом автора для
  уточнений от родни. Это **живой документ**, не итог: «снимок памяти на дату».
- В книге нет выдуманного: где архив молчит — пишем «обстоятельства
  неизвестны», а не сочиняем.
- **Персональные данные.** Сканы родословной, лица людей, имена и даты живых
  родственников — это персональные данные. По умолчанию `sources/*.pdf` и
  `assets/portraits/` можно держать вне git (см. `.gitignore`). Публикуйте
  только то, на что есть согласие родни.
- Git push в публичный репозиторий с персональными данными выполняете **вы
  сами** (агент намеренно не пушит чужие перс. данные за вас).
